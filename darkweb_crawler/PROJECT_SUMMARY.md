# 📋 프로젝트 요약

## 개요
단일 .onion 도메인을 분석하는 **어니언 도메인 분석 에이전트** 

### 목표
- 🎯 사용자가 제공한 하나의 .onion 도메인에 대해 **종합 보안 분석** 수행
- 📊 접근성, 색인, 콘텐츠 분석 기반 신뢰도 점수 산출
- 📄 자동화된 HTML 보고서 생성

---

## 핵심 기능

### 1단계: 접근성 검증
- **상태코드 확인**: 0, 200, 403, 502 등 HTTP 응답 분석
- **응답 시간 측정**: 활성도 판단
- **HTML 수집**: 성공 시 콘텐츠 분석용 HTML 저장

### 2단계: 색인 확인
- **Ahmia 검색**: 검색 결과 건수 확인
- **DuckDuckGo 검색**: 색인 여부 확인 (True/False)

### 3단계: 콘텐츠 분석 (HTML 수집된 경우만)
- **불법 콘텐츠 탐지**: 마약, 무기, 문서 위조 등
- **신뢰도 %**: 0-100% 범위의 판정 신뢰도

### 4단계: 카테고리 분류 (HTML 수집된 경우만)
- **사이트 타입 분류**: 포럼, 마켓플레이스, 시장 정보 등
- **주요/부가 카테고리**: 다단계 분류 지원

### 5단계: 신뢰도 점수 계산
```
신뢰도 = 접근성(40점) + 색인(30점) + 콘텐츠(30점)
최대 100점
```

### 6단계: 보고서 생성
- **HTML 형식**: 차트 + 통계 + 상세 정보
- **자동 저장**: analysis_reports/ 폴더에 저장

| 계층 | 역할 | 구성 |
|------|------|------|
| **입력부** | 도메인 입력 | CLI 또는 웹 인터페이스 |

| **분석부** | 콘텐츠 분석 | 로컬 분석 모듈 |

## 시스템 구성

### 로컬 (macOS)
```
agent.py → 분석 시작
    ↓
launcher.py → CLI 또는 웹 모드 선택
    ⬇
analyzer/ + reporters/ → 로컬 분석
    ↓
analysis_reports/ → HTML 보고서 저장
```

### 원격 (Ubuntu 서버)
```
Flask app.py (5000 포트)
    ↓
Tor socks5h (9050 포트)
    ↓
.onion 도메인 접근
```

## 실행 흐름

```
1. 도메인 입력 (CLI 또는 웹)
   ↓
2. 서버 요청 (POST /api/analyze_domain)
   ├─ 접근성 검증 (HTTP 요청)
   ├─ 색인 확인 (Ahmia, DuckDuckGo)
   └─ HTML 크롤링 (상태코드 0 제외)
   ↓
3. 로컬 분석 (HTML 수집된 경우만)
   ├─ 콘텐츠 분석
   ├─ 카테고리 분류
   └─ 신뢰도 계산
   ↓
4. 보고서 생성
   └─ analysis_reports/[domain]_[timestamp].html
```

## 주요 설정

### DDOS 필터 우회
```python
DDOS_FILTER_MAX_RETRIES = 5          # 최대 5회 재시도
DDOS_FILTER_RETRY_DELAY = 30         # 30초 대기 후 재시도
USE_ROTATING_USER_AGENTS = True      # User-Agent 회전
TOR_IDENTITY_CHANGE_ENABLED = True   # Tor IP 변경 (포트 9051)
```

### Access Queue 우회
```
15초 대기 후 "/" 경로 3회 재요청
(50KB+ HTML 수집까지 계속 시도)
```

## 에러 처리

| 상황 | 처리 방법 |
|------|---------|
| **상태코드 0** | HTML 수집 건너뜀 (분석 불가 표시) |
| **상태코드 403** | User-Agent 회전 + Tor 식별성 변경 + 재시도 |
| **Access Queue** | 15초 대기 후 3회 재요청 |
| **Connection Error** | 3회 재시도 후 포기 |
